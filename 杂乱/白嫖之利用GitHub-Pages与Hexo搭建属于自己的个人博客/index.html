一、前言

为什么会写这样一篇文章呢？Hexo 的官方文档已经很棒了；网上对于 Hexo 的文章也已经有很多了，但是很多文章都是基于 2.* 的版本加上有些文章在自定义域名有错，所以写了这篇文章算是对官网文档的小部分补充吧。

七、启用文章评论功能

文章的评论功能主要依赖 Hexo 插件，主要有：gitalk、disqus等等。因为有的主题会整合部分插件，所以使用不同的主题，开启评论功能的方法有所不同，这里依然以 fluid 主题与 gitalk 举例。首先依旧使用 npm 下载 gitalk 包，然后前往 GitHub 创建一个 OAuth App，如下图

Homepage URL 填写网站的地址，这里好像只能填写为图中那样，直接填写自定义的域名好像不行，还望大佬指点。图中的地址是启用 HTTPS 的，如何启用见文章第九步。然后打开 fluid 文件夹的配置文件启用评论功能，然后进行下一步配置

gitalk 利用的是 GitHub 仓库的 Issues 作为评论区的，所以每次需要管理员手动初始化。如果你的主题没有整合评论，可以参考这个插件 hexo-plugin-gitalk

八、图文混排

这里我选择了使用 文章资源文件夹 在文章中引用图片需要修改为 ![图片名字](\图片名字) ，例如我想引用一个名为 test.jpg 的图片就需要这样 ![test](\test.jpg) 。原因在于 Hexo 推送时会将图片与文章一起放在一个文件夹中推送到仓库中，所以图片与文章其实在同一个目录下。由于需要使用相对路径，就造成了使用 Typora 写文章没有图片预览，希望有大佬指导一下😂。

九、自定义域名及启用HTTPS

如果我们已经有了一个域名，直接进入服务商的域名解析设置，这里以阿里云举例。首先我们进入解析列表添加一个 CNAME 记录，主机记录填写为 @ 或者其他自己想要的，记录值改为 用户名.github.io ，详情见下图

然后进入 Hexo 的 source 文件夹下添加一个 CNAME 文件，不要任何后缀 ，填入自己的网址，见下图。这里其实是 Github Pages 的要求，当初一直以为是 Hexo 的要求😂

然后进入 GitHub 的仓库设置页面添加自己的域名，如果 GitHub 能成功解析，会出现红框里的解析结果，勾选上 Enforce HTTPS 会启用 HTTPS。

记得将 Hexo 部署到 GitHub 上，就能通过自己的域名访问啦！

十、总结

白嫖之旅到此结束了，Hexo 作为一个静态博客，自己感觉还是部署还是十分简单、界面优美，关键是全套的白嫖怎么不香呐！最后欢迎光临 sofiworker的个人博客
